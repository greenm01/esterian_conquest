# Esterian Conquest Release Artifacts

This directory holds the current downloadable Esterian Conquest release artifacts.

## EC v1.5 DOS Bundles

These DOS zip bundles are for emulator testing and operator reproduction.

| Bundle | Executables | DOSBox-X | dosemu2 | Notes |
|---|---|---|---|---|
| `ec-v1.5-classic.zip` | Original packed DOS executables | Verified | Not currently working | 8s smoke pass from `/tmp` with a real game dir and known-good local-console `CHAIN.TXT`. |
| `ec-v1.5-unlocked.zip` | Curated runnable plain-MZ executables from `NC_UNLOCKED/` | Verified | Not currently working | 8s smoke pass from `/tmp`; current `ECGAME.EXE` is rebuilt from `ECGAMEU.EXE` with corrected MZ size fields. |

Here, `Verified` means the current package survived the repo's DOSBox-X smoke launch from `/tmp` without the old INT 6 / GPF failures. DOSBox-X is currently the only verified local runner for the original EC v1.5 binaries.

Both archives are generated by:

```bash
python3 scripts/build_release_packages.py --verify
```

The unlocked package build refreshes `NC_UNLOCKED/` first via:

```bash
python3 tools/unlzexe/rebuild_unlocked.py --verify
```

The bundled game directory is built from the preserved `fixtures/ecutil-init/v1.5` baseline so package generation does not depend on mutable working copies under `original/v1.5/`.

## nc-connect Player Archives

The player-facing Rust public download is `nc-connect`.

| Bundle | Platform | Contents |
|---|---|---|
| `nc-connect-v1.0.0-beta.1-windows-x64.zip` | Windows x64 (`x86_64-pc-windows-msvc`) | `nc-connect.exe`, player manual PDF, `README.md`, `BUILD-INFO.txt`, and font license files |
| `nc-connect-v1.0.0-beta.1-linux-x64.tar.gz` | Linux x64 (`x86_64-unknown-linux-gnu`) | `nc-connect`, player manual PDF, `README.md`, and `BUILD-INFO.txt` |
| `nc-connect-v1.0.0-beta.1-macos-arm64.tar.gz` | macOS Apple Silicon (`aarch64-apple-darwin`) | `nc-connect`, player manual PDF, `README.md`, and `BUILD-INFO.txt` |

They are generated by:

```bash
python3 scripts/build_playtest_bundle.py --artifact nc-connect --verify
```

Windows release packaging is standardized on native MSVC builds
(`x86_64-pc-windows-msvc`). The Windows GUI archive includes only
`nc-connect.exe`; `nc-connect-cli` is Cargo/source-only and is not bundled in
the public GUI player zip.

The combined Rust private-beta bundle remains a local/internal helper. It is
not the public player download.

See [docs/release-policy.md](../docs/release-policy.md) for the current
distribution policy.
