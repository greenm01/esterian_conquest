# Esterian Conquest Release Artifacts

This directory holds the current downloadable Esterian Conquest release artifacts.

## EC v1.5 DOS Bundles

These DOS zip bundles are for emulator testing and operator reproduction.

| Bundle | Executables | DOSBox-X | dosemu2 | Notes |
|---|---|---|---|---|
| `ec-v1.5-classic.zip` | Original packed DOS executables | Verified | Not currently working | 8s smoke pass from `/tmp` with a real game dir and known-good local-console `CHAIN.TXT`. |
| `ec-v1.5-unlocked.zip` | Curated runnable plain-MZ executables from `EC_UNLOCKED/` | Verified | Not currently working | 8s smoke pass from `/tmp`; current `ECGAME.EXE` is rebuilt from `ECGAMEU.EXE` with corrected MZ size fields. |

Here, `Verified` means the current package survived the repo's DOSBox-X smoke launch from `/tmp` without the old INT 6 / GPF failures. DOSBox-X is currently the only verified local runner for the original EC v1.5 binaries.

Both archives are generated by:

```bash
python3 scripts/build_release_packages.py --verify
```

The unlocked package build refreshes `EC_UNLOCKED/` first via:

```bash
python3 tools/unlzexe/rebuild_unlocked.py --verify
```

The bundled game directory is built from the preserved `fixtures/ecutil-init/v1.5` baseline so package generation does not depend on mutable working copies under `original/v1.5/`.

## Rust Playtest Bundles

The Rust playtest bundles package the public `ec-game` / `ec-sysop` binaries
plus the minimum supporting assets needed for real campaign playtesting without
a Rust toolchain.

| Bundle | Platform | Contents |
|---|---|---|
| `esterian-conquest-v1.0.0-beta.1-linux-x64.tar.gz` | Linux x64 (`x86_64-unknown-linux-gnu`) | `ec-game`, `ec-sysop`, PDF manuals, themes, sample `config.kdl`, quick-start `README.md`, and `BUILD-INFO.txt` |
| `esterian-conquest-v1.0.0-beta.1-macos-arm64.tar.gz` | macOS Apple Silicon (`aarch64-apple-darwin`) | `ec-game`, `ec-sysop`, PDF manuals, themes, sample `config.kdl`, quick-start `README.md`, and `BUILD-INFO.txt` |

They are generated by:

```bash
python3 scripts/build_playtest_bundle.py --verify
```

Explicit macOS examples:

```bash
python3 scripts/build_playtest_bundle.py --target aarch64-apple-darwin --verify
```

### macOS CLI note

The macOS archives contain command-line binaries, not `.app` bundles. If
Gatekeeper blocks them after download, remove the quarantine attribute from
the unpacked bundle root:

```bash
xattr -d com.apple.quarantine ./bin/ec-game ./bin/ec-sysop
```
